  • Janeiro/2025: Planejamento e Pesquisa Inicial
    • Semana 1-2: Definição detalhada e escopo do projeto com base nos dados disponíveis;
    • Semana 3-4: Levantar as informações sobre bibliotecas, ferramentas e datasets relevantes para trabalhar com Libras;

  • Fevereiro/2025: Coleta e Integração de Dados
    • Semana 1-2: Reunir bases de dados existentes, como as do INES, UFPE e SignBank;
    • Semana 3-4: Integrar as fontes de dados em um Cross-Dataset, aplicando limpeza e padronização de dados, juntamente com o dataset próprio criado;

  • Março/2025: Pré-processamento dos Dados
    • Semana 1-2: Realizar amostragem de frames e aplicar técnicas de Data Augmentation;
    • Semana 3-4: Implementar a estimação de landmarks usando a biblioteca MediaPipe;

  • Abril/2025: Desenvolvimento do Modelo
    • Semana 1-2: Desenvolver uma arquitetura inicial de CNN para extração de características visuais;
    • Semana 3-4: Integrar a rede LSTM para lidar com dados sequenciais, com base no protocolo experimental desenvolvido;

  • Maio/2025: Treinamento e Validação
    • Semana 1-2: Treinar o modelo utilizando o Cross-Dataset criado e avaliar a eficácia em dados de validação;
    • Semana 3-4: Ajustar hiper parâmetros e otimizar a performance do modelo;

  • Junho/2025: Testes e Ajustes
    • Semana 1-2: Realizar testes externos em dados não vistos durante o treinamento, validando a generalização do modelo;
    • Semana 3-4: Implementar ajustes baseados nos resultados dos testes, considerando as análises apresentadas;

  • Julho/2025: Interface do Usuário
    • Semana 1-2: Desenvolver uma interface inicial para demonstração do sistema de tradução contínua;
    • Semana 3-4: Integrar funcionalidades bidirecionais (gestos para texto/fala e vice-versa);

  • Agosto/2025: Testes em Cenários Reais
    • Semana 1-2: Aplicar o sistema em cenários reais, como interações em ambientes educacionais e institucionais;
    • Semana 3-4: Coletar feedback dos usuários e realizar melhorias no sistema;

  • Setembro/2025: Documentação e Finalização
    • Semana 1-4: Revisão e refatoração de códigos se necessário;

  • Outubro/2025: Documentação e Finalização
    • Semana 1-2: Criar a documentação técnica e o guia do usuário;
    • Semana 3-4: Preparar materiais para publicação do sistema por meio de repositórios e apresentações;

  • Novembro/2025: Entrega e Divulgação
    • Semana 1-2: Realizar a apresentação do sistema;

  • Dezembro/2025 / Janeiro/2026: Entrega e Divulgação
    • Semana 1-2: Terminar de escrever a dissertação;
    • Semana 3-4: Publicar os resultados e compartilhar com a comunidade acadêmica;
